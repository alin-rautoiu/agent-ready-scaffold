# Dashboard and UX Knowledge

Stable UX guidance for data-heavy admin pages.

## Decision-First Hierarchy

- Dashboard and list-heavy admin pages should expose one primary operational signal first, then secondary diagnostics.
- UI changes on dashboard and queue pages should include explicit scanability checks: label clarity, units, ordering, and mobile first-viewport readability.

## Visual Choice Discipline

- Choose charts and summary visuals by analytical task type, not by style preference.
- Metrics with operational implications should include local reference context such as target, threshold, or baseline.

## Practical Implications

- Dense admin pages should optimize for fast triage, not ornamental variety.
- Mobile views must preserve the top signal and next action without forcing deep scrolling.
- Table and card treatments should help users compare, rank, or act — not merely display data.
