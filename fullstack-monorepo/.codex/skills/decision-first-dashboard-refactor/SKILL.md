# Decision-First Dashboard Refactor

Use this when a dashboard or admin page is dense but unclear.

## Method

1. Identify the top 1-3 user decisions.
2. Map each decision to one primary metric and one next action.
3. Reorder the interface so the primary signal appears first.
4. Add concise labels, units, and period or status context.
5. Verify that the mobile first viewport preserves the top signal.
