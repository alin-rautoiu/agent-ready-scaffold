## Microcopy Defaults

- Button labels must describe the immediate action, not a generic submission.
- Headings must name the page, object, or action specifically enough to stand alone.
- Error messages must say what failed, where it failed, and how to recover.
- Empty states must distinguish no data from no matching results and provide a next action when one exists.
- Toasts must state what happened.

## Trust-Sensitive UX

- Make irreversible actions explicit about consequence, scope, and reversibility.
- Do not use manipulative consent language or ambiguous opt-in/opt-out text.
- Keep user choices transparent and reversible where the product allows it.
- Avoid exposing secrets or credential values in the UI or error states.

- Clarify lifecycle states and status transitions.

## Confirmation And Notification Copy

- Confirmation dialog titles should include the action and object identity.
- Confirmation bodies should include consequence and affected scope.
- Primary action text should repeat the concrete action; secondary action should preserve the safe path.
- Success copy should include the affected object when useful.
